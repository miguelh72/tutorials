# Git Pro Book
Folder will include code used to learn how to use basic Git functions as described in the Git Pro Book (https://git-scm.com/book/en/v2), such as:

  * init - Create skeleton repository in current directory.
  * clone - Clone repository from a server, usually giving a url to repository as parameter and using https protocol.
  * add - Add file to staging area. Can be used to begin to track a file or stage modified file.
  * status - Display status (untracked, unmodified, modified, or staged) of each file not being ignored by .gitignore rules.
    - --short or -s modifier - display compact status. Uses two column descriptor: left-hand column indicated staging area status, and right-hand column indicated working directory status. "?" symbolizes untracked; "A" symbolizes newly tracked file; "M" symbolizes modified file; "R" symbolizes renamed file.
